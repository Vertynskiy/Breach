IMPLEMENTATION.md# Breach - Implementation Status\n\n## Project Overview\nBreach is a psychological management simulator where players survive 20 days at an abandoned research station while managing resources, equipment, sanity, and anomalous events.\n\n## Completed Components\n\n### Core Systems\n- **GameState** - Central game state management\n- **ResourceManager** - Tracks fuel, food, water, parts, batteries\n- **SanitySystem** - Manages player mental state with 4 stages (stable, anxious, panicked, fractured)\n- **EventGenerator** - Generates random story events and anomalies\n- **TimeManager** - Handles day/night cycle and in-game time\n- **SaveManager** - Game save/load functionality\n\n### UI Systems\n- **ScreenManager** - Screen switching and rendering\n- **BaseScreen** - Abstract base for all screens\n- Screen implementations:\n  - ObservationScreen\n  - ControlPanelScreen\n  - MonitorsScreen\n  - LaboratoryScreen\n  - JournalScreen\n- **UIElements** - Button, TextDisplay, Panel classes\n\n### Configuration\n- **settings.py** - All game constants and configurations\n  - Screen dimensions (1280x720)\n  - Color palette\n  - Resource configurations\n  - Difficulty settings\n  - Sanity mechanics\n  - Repair costs\n\n## File Structure\n```\nBreach/\n├── main.py                  # Game entry point\n├── settings.py              # All game constants\n├── src/\n│   ├── core/\n│   │   ├── __init__.py\n│   │   ├── game_state.py    # Central game state\n│   │   ├── resource_manager.py\n│   │   ├── sanity_system.py\n│   │   ├── event_generator.py\n│   │   ├── time_manager.py\n│   │   └── save_manager.py\n│   └── ui/\n│       ├── __init__.py\n│       ├── screen_manager.py\n│       ├── ui_elements.py\n│       └── screens/  # Future screen implementations\n└── saves/               # Game save directory\n```\n\n## Key Features\n\n### Resource Management\n- Fuel: Powers generator (critical for survival)\n- Food: Sustains player health\n- Water: Essential resource\n- Parts: For equipment repair\n- Batteries: Powers auxiliary systems\n\n### Sanity System\n- **Stable (0-30)**: Normal state\n- **Anxious (31-60)**: Mild stress\n- **Panicked (61-80)**: High stress, visual effects possible\n- **Fractured (81-100)**: Critical - Game Over if sustained for 10 minutes\n\n### Day/Night Cycle\n- 20 in-game days to survive\n- Day: 8:00 - 20:00\n- Night: 20:00 - 8:00\n- Different resource consumption rates\n- Events triggered based on time and conditions\n\n### Difficulty Levels\n1. **Normal**: Baseline consumption, 5.5 events/day\n2. **Hard**: 20% more resource consumption, 7 events/day\n3. **Insane**: 40% more consumption, 9 events/day, 85% anomaly chance\n\n## Game Loop\n```python\nwhile running:\n    # Event handling\n    for event in pygame.event.get():\n        screen_manager.handle_event(event)\n    \n    # Update game state\n    delta_time = clock.get_time() / 1000.0\n    game_state.update(delta_time)\n    screen_manager.update()\n    \n    # Render\n    screen.fill(COLOR_BLACK)\n    screen_manager.render(screen)\n    pygame.display.flip()\n    clock.tick(FPS)\n```\n\n## Next Steps for Completion\n\n1. **Enhanced Screen Implementations**\n   -
